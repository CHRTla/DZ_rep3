# Инструкция #
## Работа с текстом ##
* **Жирный текст** - **
* *Курсивный текст* - *
* ~Зачеркнутый текст~ - ~
* Выделение заголовков - # в начале строки
* Ненумерованные списки - *(или -) в начале строки и пробел
* Нумерованные списки обозначаются обычными цифрами 1,2,3
> Цитирование. Обозначается знаком > в начале текста
* Добавление изображений: !+[Описание картинки]+(адрес картинки)
![Текст с описанием картинки](https://wallbox.ru/resize/320x240/wallpapers/main/201547/9dbdf1269bf5a53.jpg)
* Чтобы построить таблицу, можно воспользоваться вертикальными линиями, а для добавления заголовка - вставить пунктирную линию после первой строки. Можно выровнить столбцы с помощью двоеточий (после пунктирных линий)
* Для того чтобы создать горизонтальную линию необходимо поместить три (или более) дефиса или звездочки на отдельной строке текста
* Упрощенный способ создания гиперссылки.Для этого достаточно поместить URL-адрес или почтовый адрес в угловые скобки, и Markdown сделает его гиперссылкой.
* Подстрочные символы - в начале sub (вокруг знаки сравнения), далее текст и /sub (вокруг знаки сравнения)
* Надстрочные символы - в начале sup (вокруг знаки сравнения), далее текст и /sup (вокруг знаки сравнения)

## Команды Git ##
* Инициализация локального репозитория: git init
* Просмотр историй версий: git log
* Добавление файла: git add
* Проверка версии: git version
* Просмотр изменений: git diff
* Проверка статуса: git status 
* Создание коммита: git commit -m "текст"
* Переход к коммиту: git checkout
* Отмена изменений: git reset
* Удаление файлов из индекса и рабочей копии: git rm
* Удаление мусора из рабочего каталога: git clean
* Отображение встроенной документации: git help
* Отправка подтвержденных изменений в удаленный репозиторий: git push
* Отображение метаданных автора, связанных со строками, которые были внесены в файл при коммите: git blame
